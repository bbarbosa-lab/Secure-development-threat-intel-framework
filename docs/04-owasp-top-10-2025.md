# 04 - OWASP Top 10 2025 with Focus on APIs and Threat Modeling

This document provides a practical view of the **OWASP Top 10 2025**, with emphasis on web applications and APIs, and explains how to use threat intelligence to prioritize and mitigate these risks more effectively.

## Overview of OWASP Top 10 2025

The OWASP Top 10 is the industry-standard list of the most critical security risks to web applications and APIs, based on real-world data collected from thousands of applications.

## Detailed Analysis with Threat Intelligence Focus

### A01:2025 – Broken Access Control (Highest Risk for APIs)

**Why it remains the most dangerous risk:**

- It is the most exploited risk in multi-tenant APIs and modern applications.
- Modern frameworks make rapid development easy but often leave authorization gaps if not implemented correctly.
- Broken Object Level Authorization (BOLA) continues to be one of the most common API vulnerabilities.

**How Threat Intelligence helps:**

- **MITRE ATT&CK techniques:** T1078 (Valid Accounts), T1134 (Access Token Manipulation), T1552 (Unsecured Credentials)
- During threat modeling, ask: *"If an attacker obtains valid low-privilege credentials, which resources can they access?"*
- Monitor for anomalous access patterns (e.g., one token accessing thousands of resources in a short time).

**Priority Controls:**

- Enforce server-side authorization on **every** endpoint.
- Use non-sequential identifiers (UUIDs) for resources.
- Implement automated access control matrix testing.

### A03:2025 – Software Supply Chain Failures

**Why it gained prominence in 2025:**

- Supply chain attacks (SolarWinds, XZ Utils, Codecov, etc.) have become more frequent and severe.
- Organizations increasingly rely on third-party components, CI/CD pipelines, and package registries.

**How Threat Intelligence helps:**

- Use **NVD + EPSS** to prioritize vulnerabilities that are actively being exploited.
- Threat feeds (MISP, Talos) can indicate when a specific dependency or build tool is targeted in active campaigns.
- Monitor for indicators of compromised packages or unusual behavior in build pipelines.

**Priority Controls:**

- Automated Software Composition Analysis (SCA) in CI/CD.
- Generate SBOMs for every release.
- Implement code signing and provenance attestation for critical artifacts.

### A05:2025 – Security Misconfiguration

**Why it is so common:**

- Default configurations in frameworks and cloud providers prioritize ease of use over security.
- Many teams lack a formal hardening process.

**How Threat Intelligence helps:**

- SANS and CIS publish alerts about configurations actively being exploited.
- Threat feeds indicate when new misconfiguration exploitation techniques are observed in the wild.

**Priority Controls:**

- Use **CIS Benchmarks** as the baseline for the entire stack.
- Disable debug mode and unnecessary features in production.
- Remove headers that leak unnecessary information.
- Automate hardening through Infrastructure as Code (IaC).

### A06:2025 – Vulnerable and Outdated Components

**Why it remains critical:**

- The volume of published CVEs continues to grow.
- Dependency confusion and typo-squatting attacks are increasing.

**How Threat Intelligence helps:**

- Combine **NVD + EPSS** scores with active exploitation data from threat feeds.
- MISP and commercial feeds can show when a specific CVE is being used in ongoing campaigns.

**Priority Controls:**

- Automated SCA in the CI/CD pipeline.
- Maintain an up-to-date Software Bill of Materials (SBOM).
- Prioritize remediation using CVSS + EPSS + real-world exploitation context.

## How to Prioritize OWASP Top 10 in Your Context

Not all risks carry the same weight for every application.

### Prioritization Recommendations

| Application Type                    | Highest Priority Risks      | Justification |
|-------------------------------------|-----------------------------|-------------|
| **Public / Multi-tenant APIs**      | A01, A05, A07, A03          | Large attack surface + high impact of unauthorized access |
| **Internal Corporate Web Apps**     | A02, A06, A04               | Lower external exposure but high risk from misconfiguration and vulnerable components |
| **Microservices / Cloud Native**    | A01, A03, A08               | Complex service-to-service communication + supply chain risks |
| **Applications with Highly Sensitive Data** | A01, A04, A07, A09     | Data breach impact is critical |

## Integrating OWASP Top 10 with MITRE ATT&CK

This integration is one of the most powerful aspects of a threat-informed approach.

| OWASP Risk                        | Most Relevant ATT&CK Techniques | Threat Modeling Question |
|-----------------------------------|----------------------------------|--------------------------|
| A01 – Broken Access Control       | T1078, T1134, T1552             | How could an attacker with valid low-privilege credentials escalate access? |
| A03 – Software Supply Chain       | T1195, T1608                    | If this dependency or pipeline component is compromised, what would be the impact? |
| A05 – Security Misconfiguration   | T1059, T1608                    | Which default configurations in this framework/cloud are exposed, and how would an attacker exploit them? |
| A06 – Vulnerable Components       | T1195                           | If this dependency is compromised, what is the potential blast radius? |
| A07 – Authentication Failures     | T1078, T1110, T1552             | How could an attacker obtain valid credentials or bypass authentication? |
| A08 – Software & Data Integrity   | T1195, T1608                    | How could an attacker compromise our build pipeline or release artifacts? |

## Practical Recommendations

1. **Do not treat the OWASP Top 10 as a generic checklist.** Use it as a starting point for risk discussions tailored to your specific application and context.
2. **Always combine OWASP risks with MITRE ATT&CK** during threat modeling sessions. This makes the exercise significantly more concrete and actionable.
3. **Use threat intelligence** to decide which risks deserve immediate attention (i.e., which ones are actively being exploited right now).
4. **Document** your prioritization decisions and the controls you implement.
