# Contributing to Threat-Informed Secure Development

Thank you for your interest in improving this framework!

## How to Contribute

1. **Fork** the repository
2. Create a feature branch (`git checkout -b feature/amazing-improvement`)
3. Make your changes
4. Ensure documentation and examples remain consistent
5. Open a Pull Request

## Focus Areas

We especially welcome contributions in:

- New mappings between frameworks (SSDF, OWASP, ATT&CK, etc.)
- Additional practical code examples (other languages/frameworks)
- Improvements to threat intelligence integration patterns
- Translations (especially Portuguese)

## Code of Conduct

Be respectful, professional, and focused on helping developers build more secure systems through better thinking and tooling.

## Questions?

Open an issue with the label `question` or `discussion`.
